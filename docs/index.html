jQuery Ajax Calendar (jac) v0.0001

jac comes with one example calendar. jac does not care what calendar you use or how it's laid out as long as you have a few classes on a few key elements. The example calendar shows you all the basics and then some.

Initialization


								$( '#calendar' ).jac({ options });
If we made it any easier, you wouldn't sleep at night charging your clients for this functionality.

Options

Options can either be passed as a javascript literal or set as data-* attributes on the invoking element (#calendar above). Passed options always take precedence over data-*.

Required
Option Description
options.events,
options.src,
data-src=""		 One of these three must be present for events to be rendered. jac will still number dates and manage next and previous actions even if you have no events. (watch the console for an error message if none of these are present.) options.events must be a javascript object literal of events.

Defaults
Option Description
tz America/Chicago (because there isn't 'America/BigD')
The timezone is lightly used for locale right now. Use any timezone available to moment.js. Be sure to send your dates and times in UTC (1971-09-01T18:35:47Z) format so that moment() will autoconvert for tz.
startWeek 0
0..6 = Sunday..Saturday Put any weekday in there and jac will figure out and number the weeks correctly. (Hello, my muslim semitic friends!)
time moment( [options.time|'YYYY-MM-DD'] )
Pass a moment() object or a date string that can be parsed by moment on creation, e.g., moment( options.time )
weekdayFormat dddd
Use any moment() formatting option. This is one of the few things jac takes seriously. He will set the right weekday names based on the options.time and the options.startWeek.

Usage (finally!)

Generally speaking, HTML is already a templating language, the browser a templating engine. jac understands that and uses it to his advantage. You build your calendar the way you want it. Buy a kick ass calendar template from somewhere and let jac manage it. jac truly doesn't care about styling, just relationship and position. The display of things that jac controls is still under your control through a (very!) light template engine. You must have three HTML elements for jac to use: a calendar div or wrapper of some kind that will hold your data-*, your day containers, a day template and an event template. These templates can live anywhere on the page but should start hidden.
Calendar
						
    <!-- #calendar can be anything. You'll pass this element into jac's init, i.e., $( '#calendar' ).jac({ options}) -->
    <div id="calendar"
        data-tz="Australia/Lord Howe"
        data-startWeek="5"
        data-time="1971-09-01T18:35:47Z"
        data-weekdayFormat="dd"
        data-dayTpl="my-cal-tpl"
        data-eventTpl="my-event-tpl"
        data-calendarDateFormat="Mo"
        data-calendarTitleFormat="MMMM YYYY (Qo)"
        data-otherMonthClass="text-muted text-sm"
        data-highlightToday="false"
        data-todayHighlightClass="today"
        data-events="[
        {
            'id': 1,
            'title': 'Hot Dog Eating (Day)',
            'image': 'http://via.placeholder.com/68x100',
            'start_time': '2018-08-06T12:30:00Z',
            'end_time': '2018-08-06T14:30:00Z',
            'location': 'Dallas, TX',
            'description': 'See Takeru Kobayashi wrap another belt around his bulging gut!'
        },
        ]"
    >
        <!--
            .calendar-date is optional but .events is not. If you don't have .calendar-date,
            jac won't number the calendar dates but he will still manage events and prev/next
            actions for setting month timestamp. If you don't have .events, then, um, you don't have events.
            If you have multiple .events, they all get populated. This is helpful for multiview calendars.
        -->
    </div>
    <!-- This calendar markup is just an example markup; jac doesn't care if you use tables, ul, ol, div, or whatever.
         The most important things to be aware of here are the day, wkdy*, w* and d* classes -->
    <ul class="weekdays">
        <!--
            wkdy0 does NOT refer to the weekday numbering used by moment().
            It's the index of the week based on startWeek. So, if 5 == startWeek, wkdy0 = 5 and so on;
        -->
        <li class="weekday wkdy0"></li>
        <li class="weekday wkdy1"></li>
        <li class="weekday wkdy2"></li>
        <li class="weekday wkdy3"></li>
        <li class="weekday wkdy4"></li>
        <li class="weekday wkdy5"></li>
        <li class="weekday wkdy6"></li>
    </ul>
    <!--
        .days is just a class that you can use to quickly walk the DOM to this week.
        w1 refers to the week number of the calendar, NOT the month.
        Depending on startWeek, two days which might normally cross over the month week
        may end up being in the same calendar week.
    -->
    <ul class="days w1">
        <!--
            You set these classes on your calendar date containers for jac
            to use as a guide. Again, d1 does NOT refer
            to moment() weekday numbering. _startWeek changes a bunch of stuff, yo.
            jac requires .day so he knows where to search for .w*.d* classes and
            append events to. Whatever is in here when the page loads will be erased
            when the day template is loaded. You're just making containers for your
            templates here. Finally, you can just have a one week, three day calendar,
            if you want to. jac will just ignore the missing date containers.
        -->
        <li class="day w1 d1"></li>
        <li class="day w1 d2"></li>
        <li class="day w1 d3"></li>
        <li class="day w1 d4"></li>
        <li class="day w1 d5"></li>
        <li class="day w1 d6"></li>
        <li class="day w1 d7"></li>
    </ul>

    </ul>

Day Template
						
    <!-- You can set the id (cal-day-tpl is the default) for this tpl in jac's settings to identify the template. -->
    <div id="cal-day-tpl" style="display: none;">
        <!--
            .calendar-date is optional but .events is not. If you don't have .calendar-date,
            jac won't number the calendar dates but he will still manage events and prev/next
            actions for setting month timestamp. If you don't have .events, then, um, you don't have events.
            If you have multiple .events, they all get populated. This is helpful for multiview calendars.
        -->
        <span class="calendar-date col-12 text-left"></span>
        <span class="events col-12"></span>
    </div>
Event Template
						
    <!-- You can set the id (cal-event-tpl is the default) for this tpl in jac's settings to identify the template. -->
    <div id="cal-event-tpl" style="display: none;">
        <!-- The .event class is required on your single event container.  jac will set the event.id as '#event-[id]' on this container. -->
        <div class="event">
	    <!--
    	    When jac gets your events and he gets this template and starts to fill in values,
	        he will read the entire event object returned by your script or json file.
	        Then he uses every key returned as an event-* class and tries to set those values
	        for you, too.  So, for example, if you need to display something, have your json
	        return the key like {headline: 'Most Amazing Event Evah!!!111!1!!111'} and jac
	        will fill .event-headline with that text. Any classes that don't start with event- are ignored.
	    -->
            <div class="event-title"></div>
            <div class="times text-sm">
                <!-- if jac detects a time/date value to insert, he will check for a
	             desired format on the element it will display in. Use anything moment() uses. -->
                <span class="event-start_time" data-format="h:mm a"></span> - <span class="event-end_time" data-format="h:mm a"></span>
            </div>
            <div class="event-location"></div>
        </div>
    </div>
If you copied all this code into a single html page, you'd have a basic calendar that jac can manage and display events for. Hint, hint.

Event (festive, not window) Loading

Stop Hectoring me! - Ajax

You're probably here because you needed a calendar that could load events from a json source and render them nicely in your calendar. jac makes it pretty darn easy. You can set your src to be a local .json file (the file must be accessible using a URL) or a remote endpoint that returns an array of javascript objects. That's all. You can set the src in the passed options or as a data-* on the calendar. Either way, jac queries it for events by appending the date to the request. So, if you have data-src="https://your.endpoint.com/events", jac appends [?|&]jac-time=[options.time.toISOString()].

Every time you call $( '#calendar' ).jac();, jac requeries for events and renders the entire calendar. So, if you want to change your src, get next months events and render them 


    $( '#calendar' ).data( 'src', 'http://your.otherdomain.com/archived_events' ).data( 'time', moment().add( 1, 'months' ) ).jac();
If your src sends events outside of the time display window, they are ignored, so using one monolithic json file is possible, if you want. You can also poke your eyes out with a bacteria-laden, craft-glued, tongue depressor, but I don't recommend that, either.

Passing events in the init object

This will "break" if you are letting your users move forward and back in the calendar. The calendar will be rendered and the dates will be created according to the rules, but as soon as you switch months, the calendar will be empty of events. Mostly, you want to update the data-src on the calendar object with new events to read if you just really have to use a file.

All the available and required classes are listed below. Not all classes are required (for instance, you don't have to have prev/next buttons)
Class Scope/Container Set By Description
.jac-calendar calendar jac jac sets this class to help him find the calendar containers easier. If no #id is set on the calendar container, jac uses this class to find the calendar and set an #id. Don't remove it.
.calendar-title Any You Based on your value for data-calendar-title-format=*, this class container will be filled with the formatted calendar title. See moment.js docs for formatting dates and passing escaped data for things like making linked titles.
.cal-btn-prev Any You Regresses calendar's data-time="* by one month and gets events (festive, not window). (Set current time's month date to 1 and then subtract 1 month) Set this class on any HTML element that you want to use as a clickable control. When jac starts up, he sets a click handler on all elements with this class. This allows you to have these controls in multiple places like top and bottom, scrollable div, flyout menu, etc.
.cal-btn-next Any You Advances calendar's data-time="* by one month and gets events (festive, not window). (Set current time's month date to 1 and then add 1 month) Set this class on any HTML element that you want to use as a clickable control. When jac starts up, he sets a click handler on all elements with this class. This allows you to have these controls in multiple places like top and bottom, scrollable div, flyout menu, etc.
.cal-btn-today Any You Sets calendar's data-time="* to the current date and gets events (festive, not window). ( just a moment() ) Set this class on any HTML element that you want to use as a clickable control. When jac starts up, he sets a click handler on all elements with this class. This allows you to have these controls in multiple places like top and bottom, scrollable div, flyout menu, etc.
.day Any You jac will fire the select:day trigger for any element with this class. In conjunction with .w[1..6] and .d[1..7], you can create a single day display and trigger container. If you have a container that only has .w* and .d* classes, that container will not fire the select:day trigger. This allows you to a) remove the option of clicking a day to enter a new event (festive, not window) or b) to have separated entry controls from the calendar display itself.
.w[1..6] Any You jac will count off the days up to 6 weeks when numbering the calendar. This ensures there are enough day containers to display a 5 week month. 1..6 refers to week counting, but not calendar week as used by moment.js.
.w[1..7] Any You jac will insert the template found at data-day-tpl=* into this container. The day template must live in the same page as the calendar.
.jac-[YYYYMMDD] .w[?].d[?] jac jac adds this date to the day container for the corresponding .w[1..6].d[1..7] container AFTER everything is set and configured correctly for the day. If this class is not set, jac didn't find the day classes for that day. This allows you to easily query into events for a particular day. 

    var queryDate = '20180805';
    $( '.jac-' + queryDate + '.events' ).each(
        function()
        {
            // Do something awesome with all these events!
        });
.events .w[1..6].d[1..7] You This class must be on a contianer for jac to fill in the events into. Make sure you add this class to your data-event-tpl=* so that when jac inserts it, the events container already exists.
.event .events You You must have this class on the container for an individual event. jac will look in this container for all .event-* to populate from the event object returned from the json source. jac will also set the #event-[1..] based on the event-id passed in the json object on this container.
jac tells you alot about what he's doing as he does it if you care to know. All triggers are fired from the body tag to make it easier to listen to. An example of listening for custom events is below but you can also check out the jquery documentation. 

    $( 'body' ).on( '[trigger scope]', function( _windowNotFestiveEvent ) { console.log( _windowNotFestiveEvent ); } );
Trigger Scope Scope/Container Description
select:day body jac will set this event to fire when the attached container is clicked.
select:event body jac will set this event to fire when the attached container is clicked.
goto:prev body jac will set this event to fire when the attached container is clicked.
goto:next body jac will set this event to fire when the attached container is clicked.
goto:today body jac will set this event to fire when the attached container is clicked.
number:date body Fires after date is numbered and .w?.d? is completed. Returns the selector, the date and whether the container isToday.
render:event:before body Fires after all manipulations have been performed on the prepared event object but before insertion. Returns the manipulated event json object.
render:event body This event fires after the data-event-tpl=* is loaded and merged with the event json object at the moment of insertion.
render:event:after body Fires after the insertion of the data-event-tpl=*. Returns the manipulated event json object.
